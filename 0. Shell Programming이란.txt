<Shell Programming>

1. 쉘
    사용자와 커널간 인터페이스 역할
    사용자로 받은 명령을 해석하여 명령이 정확한지 확인 후 명령 처리(ls -l 등)


2. 커널
    사용자와 쉘로 부터 들어온 요청에 따라서 하드웨어를 돌리고 처리를 하는 역할


3. 쉘 스크립트
    리눅스/ 유닉스 쉘에 의해 실행되도록 설계된 프로그램


4. 쉘 스크립트 파일
    - 내장 명령
    - 함수
    - 변수
    - 문법(조건문, 반복문 등)
    - 리눅스 명령
    등에 명령들이 들어가 있는 파일
    
    인터프리터 언어로 텍스트 파일로 저장됨/ 쉘에 의해 실행됨
    읽기와 실행권한을 부여하는것을 원칙으로 한다


5. 쉘 배치 파일
    명령어들을 순차적으로 작성해 놓은 텍트스 파일
    => 주로 반복적인 작업이나 시스템 설정을 자동화 할때 사용
    => 확장자는 .sh이다 ex) test.sh
    => 쉘 배치 파일을 쉘스크립트라고도 부른다
    => 반복적인 작업을 파일로 만들어두고 파일을 실행하는 식으로 사용
    => 


6. 쉘 프로그래밍
    쉘의 내장된 기능 및 리눅스 명령어들을 사용한 프로그램 만들기


7. 주요 사용목적
    효율적인 시스템 관리
    반복작업 최소화를 위해 사용
